What Are Wireframes

Simply, Wireframes are essentialy the web developer's blueprint. Before you begin to code your webpage, it is critical to "wireframe" every page you're going to have. Planning the fonts, layout, color, ect.. This blueprint helps you and you're team to visually see what the end results shoud look like. Essentially, we use wireframes to design and visually our webpages before actually coding it.

Git Workflow

Git is crucial for team collaboration when programming. When collaborating with a group, obviously the first step for your team to do, is let the project manager create the essential git respository. From there, the project manager or whoevers in charge with the respository will share the respos. to their team so they can collaborate. Each developer should create Branches. Branches are basically the developer's version of the project. They do code strictly on their branch as it organize/doesn't mess up the origin master branch. When a developer is satisfied on their code, they can do a pull request. A pull request is basically pushing developer's branches code into th original/master which is basically where all working features should be implemented.

Ways to Create Responsive Web Design

  • CSS FLEXBOX

    • Adaptive way to create a responsive web design. Uses CSS and display:flexbox to adjust webpage design that differes in screen resolution.

  • media queries

    • CSS media queries are basically rewriting CSS files for targeted media screen width and height.

  • ViewPorts

    • View Ports are basically what the view from your browser. It's a CSS property that adjust size according to the computer's port view (monitor)

  • em's

    • Another Way to adaptively change the html element's size for a responsive web. It uses em's measurement instead of px, which helps create a responsive web design
